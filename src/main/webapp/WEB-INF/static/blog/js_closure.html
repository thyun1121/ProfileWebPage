Closure

사전적으로 풀이하면 "폐쇄"를 의미한다.

MDN에서는 아래와 같이 Closure를 정의하였다.

A Closure is the combination of a function and the lexical environment within which that function was declared.

풀이를 해보면,
“함수가 선언될 때의 lexical environment와 function의 조합” 이다.
결국 Scope를 포스팅하면서 lexical을 다뤘는데 lexical은 “코드 문맥” 자체를 의미하므로 “함수가 선언될 때 정의된 스코프, 실행 컨텍스트등의 정보와 그리고 함수의 조합” 이라고 생각하면 될 것 같다.

음… 더 풀이하이해되기 싶게 정리를 해보자면

외장함수의 실행이 종료되어도 외장함수의 변수 및 파라미터에 계속 접근할 수 있는 내장함수.

라고 정의할 수 있을 것 같다.

이렇게 말이다.

Written with StackEdit.아마 Javascript 개발을 해본 사람이라면, 아래 코드로 쉽게 이해할 수 있을 것이다.

function counter(initValue) {

    var currentValue = initValue;

    var increment = function(step) {

        currentValue += step;
        console.log('currentValue = ' + currentValue);
    };

    return increment;
}

var incrementCounter = counter(0);

incrementCounter(1);
incrementCounter(2);
incrementCounter(3);

currentValue = 1
currentValue = 3
currentValue = 6

위의 예제로 유추했겠지만, Closure를 사용함으로써의 이점은 다음과 같다.

  • private 변수 생성 및 보호
  • SideEffect 제어
  • 데이터 보존 및 활용

결국 다 같은 의미를 지니긴 한다.

위의 이점들을 토대로 예제를 하나씩 들어보자

  • Example 1
function Firms(firmName){
     var firmName = firmName;
     
     return {
          getName : function(){
               return firmName;
          }
 
          ,setName : function(changeName){
               firmName = changeName;
          }
     };
}
 
var USA = Firms("Google");
var ROK = Firms("Samsung");
 
console.log(USA.getName());
console.log(ROK.getName());
 
ROK.setName("Kakao"); 
console.log(ROK.getName());

** 이 예제에서 눈여겨볼 부분은 object로 함수를 2개 이상 return 할 수 있다는 것.

  • Example 2 (Example 1 을 prototype 방식으로 구현)
function Firms(firmName){
     this.firmName = firmName;    
}

Firms.prototype.getName = function(){
  return this.firmName
}

Firms.prototype.setName = function(firmName){
  this.firmName = firmName;
}
 
var USA = new Firms("Google");
var ROK = new Firms("Samsung");

console.log(USA.getName());
console.log(ROK.getName());

ROK.setName("Kakao");
console.log(ROK.getName());
