Javascript 기초부터 심화까지 공부해보자

Posted by thyun.ahn on February 15, 2018

Javascript

회사에서 javascript를 사용하지만 기본부터 차근히 정리하면서 공부해봤던 적은 없었기에, 이참에 정리하며 심화까지 익혀보려 한다.
선임이 참고하라며 알려줬던 어느 블로거의 포스트를 토대로 정리하며 내 나름의 요약을 해보려 한다.

1. 클래스 기반 언어 vs Javascript

1) 클래스의 정의와 인스턴스

클래스 기반 언어에서의 흐름 및 구조는 다음과 같다.

  1. 클래스 정의
  2. 정의한 클래스를 이용하여 인스턴스 생성
  3. 생성된 인스턴스를 통해 실제 액션 실현
class Alert{
 
    private String _msg;
    
    public alert( String $msg ){
        _msg = $msg;
    }
 
    public void action(){
        System.out.print( _msg );
    }
}

Alert a = new Alert( "test" );
a.action(); // test출력

위와 같이 new를 통해 인스턴스 생성 후 액션을 실현하는 코드를 "호스트 코드"라고 한다.

나같은 경우, 주로 java에서 위와 같은 방식으로 구현을 하곤 한다. 이 방식이 나로서는 익숙한 방식이지만 단점이 있다고 한다.

아무래도 실제 프로그램은 인스턴스로 인해 동작하는데, 인스턴스 생성 전 인스턴스의 동작을 정의했던 클래스의 action과 state가 정작 인스턴스 생성 후의 상황과 달라질 수 있기 때문이다.

무튼, 이러한 "호스트 코드"에 익숙한 프로그래머들은 javascript가 낯설게 느껴지게 된다는데, 그 이유는 다음 차례에서 살펴보겠다.

2) 인스턴스 밖에 없는 Javascript

제목에서 유추했겠지만, javascript에는 클래스가 없다.
java와 마찬가지로, javascript에도 ‘.’ 문법이 있다.

instance1.move(100,100);

‘.’ 구문을 통해서 클래스로부터 실체화된 인스턴스에 접근할 수 있다. 하지만 그렇다고해서 javascript에 클래스가 존재한다고 생각하면 안된다.

function move($x, $y){
	this.x = $x;
	this.y = $y;
}

var instance1 = {x:0, y:0, set:move};
instance1.set(200, 200);

위의 예제 코드의 특징을 3가지 정도로 표현하자면 아래와 같다.

  1. 클래스가 없다.
  2. move는 내부에서 this라는 변수를 사용하지만, 여전히 instance1과 아무런 관계가 없는 독립적인 함수이다.
function move($x, $y){
    this.x = $x;
    this.y = $y;
}

var instance1 = {x:0, y:0, set:move};
instance1.set(200, 200);

var rect = {x:0, y:0, width:100, height:100, pos:move};
rect.pos(100, 100);

보충설명을 하자면, java에서 함수의 소유자는 클래스이므로, 전반적으로 인스턴스나, 클래스로부터 함수가 불리는 형식이었지만, javascript에서는 이러한 구속이 없으므로, 함수에서 인스턴스를 고를 수 있는 방법도 제공하고 있음.

  1. 인스턴스는 메서드 형태로 어떤 함수를 사용하려면, 반드시 본인의 속성에 해당 함수를 특정 키와 함께 저장해야 한다.

javascript는 함수를 완전한 객체로 취급하며, 함수 그 자체도 오브젝트로부터 파생된 인스턴스이기 때문에, 함수 자체도 속성과 메서드(key-value)가 존재할 수 있다고 함.

function move($x, $y){
	this.x = $x;
	this.y = $y;
}

//그냥 키 값을 할당하면 됨!
move.test = 3;

3) 그럼 new는 뭐야?

클래스가 없는 javascript에서도 new가 사용되므로 헷갈릴 법 하다.

javascript에서 new는 오브젝트의 인스턴스를 만드는 키워드로 사용된다.
(물론, 인스턴스를 만들 때 굳이 new가 아닌 리터럴을 통해 만들 수도 있음.)

java에서 new의 역할은 다음과 같다.

  1. new 뒤에 지정된 클래스의 인스턴스를 만들고,
  2. 해당 클래스에 생성자 함수가 있는 경우, 그 함수를 호출한다.

But!!!

javascript에서의 new의 역할은 java와는 다르다.

  1. 일단 오브젝트를 하나 생성한 뒤,
  2. new뒤에 지정된 함수의 프로토타입을 방금 생성한 오브젝트의 __proto__에 참조로 잡아준다.
  3. 그런다음, 방금 생성한 오브젝트를 new 뒤의 함수에게 컨텍스트로 전달하여 호출한다.
  • new를 이용한 코드
function move($x, $y){
	this.x = $x;
	this.y = $y;
}

var instance = new move(10, 10);
  • 위 new를 이용한 코드 분석
    – 1. var instance = {};
    – 2. instance.__ proto__ = move.prototype;
    – 3. move.call(instance, 10, 10);

보아하니, new를 이용하고 바로 위와 같이 직접 사용할 수 있겠지만, 이 블로그에서는 그냥 new를 사용하라고 한다. 이유는 다음과 같다.

  1. new 구문은 c 컴파일 수준에서 한번에 처리하므로, 훨씬 고속이다.
  2. __ proto __ 속성은 브라우저에 따라 접근할 수 없는 경우(IE)도 있으므로, 애초에 불가능 할 수 있다.

그리고, new 하나만 사용하면 되는데 굳이 저렇게 3단계나 걸쳐서 쓰긴 귀찮을 것 같다…

요점을 정리하자면,

  1. javascript의 new 구문 뒤에 오는 것은 클래스가 아니라, 함수!!
  2. new를 통해 생성된 것은 오브젝트 {} !!
  3. 함수에 컨텍스트만 전달될 뿐, 생성자 함수라는 개념은 없음!!

“함수는 그저 함수입니다. 함수 내부에 this 키워드는 그저 전달된 컨텍스트를 가리킬 뿐이죠.” by. 참조 블로거.
