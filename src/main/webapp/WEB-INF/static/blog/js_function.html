Function

함수는 Javascript에서 일급 객체이며, 제일 중요한 부분으로 인식된다.
따라서 함수의 사용 사용방법 및 종류에 대해서 알아보자.

Declaration VS. Expression

앞서 Hoisting에서 Function Declaration은 hoisting이 적용되는 반면, Function Expression은 hoisting이 적용되지 않음을 설명했다.

그럼 언제 어떤 방법으로 함수를 사용하는 것이 좋을까?

사실 이 질문에 대해서는 많은 개발자들의 설왕설래가 있다. 즉, 정답은 없지만, 그래도 많이들 공감하는 것은 함수 선언식 보다는 표현식을 선호한다는 것이다.
그 이유는

  • 함수 표현식이 가독성에 유리하다.
    • 함수 선언문은 hoisting에 적용받는 관계로 가독성이 좋지 않다. 코드량이 얼마 되지 않은 경우에는 상관없지만 여러 사람이 협업하여 코드량이 5,000줄 이상일 경우, 동일 함수명을 사용하는 경우가 생길 여지가 다분하다. 이때 함수 선언식으로 동일 함수명을 사용한다면, cascading 법칙으로 나중에 hoisting되어 개발자들에게 혼란을 줄 수 있다.
  • 클로저로 사용할 수 있다.
    • 클로저는 따로 포스팅할 예정.
  • 콜백으로 많이 사용된다.
    • 주로 콜백함수는 다른 함수의 인자값으로 함수를 넘기는데, 이때 많이 사용되는 방식이 함수 표현식이다. 자세한건 아래 콜백함수에서 다루겠다.

여러곳에서 해당 내용을 찾아봐도 다들 Function Expression을 선호하는 것을 알 수 있다. 예를들어 더글라스 크락포드 도 그렇고 Airbnb javascript style guide1에서도 Function Expression을 선호한다. 그렇다고 Function Declaration을 아예 쓰지말라는 건 아니다. 다만 두 함수 사용법의 특징을 정확히 파악하고 상황별로 일관되게 쓰는 것이 중요하다.!!

Function VS. Method

Function은 주로 object 바깥에서 정의되는 반면에 Method는 주로 object 내부에서 정의된다. 그래서 Method는 보통 .이 붙어서 실행된다. obj.method() 형식으로.
Method가 object 내부에서 정의되어 obj.method()로 실행된다는 것은 this binding이 됨을 의미하며, 같은 object 내부에 있는 다른 데이터를 조작할 수 있다.

그렇다면 Function과 Method는 언제 사용하면 될까?

  • Method는 특정 객체의 action을 정의하고 실행을 하기 위해 사용된다. 이때 동일 object 내부의 data 값을 변형시키기도 하고 다른 function들과 같이 사용되기도 한다.
  • Function은 다른 context에서도 일관되게 사용이 필요할 때 주로 사용된다. 예를들면, 배열의 값들을 거꾸로 나열하는 기능이 필요할 때 이런 기능은 어떤 업무 도메인이든 상관없이 동일 기능을 필요로하므로, 이때는 Function을 사용하면 된다. (물론 각 개발자 마음이다.)

Callback Function

Javascript는 비동기형 프로그래밍이기 때문에 Callback Function을 많이 사용하며, 위에 Declaration VS. Expression에서 언급했던 것처럼 함수의 인자로 사용된다.
예를들면 이렇게

var beer = ["Sanmiguel", "StellaArtois", "Blanc", "Heineken"];
beer.forEach(function (eachName, index){
  console.log(index + 1 + ". " + eachName); 
  // 1. Sanmiguel, 2. StellaArtois, 3. Blanc, 4. Heineken
});

Callback Function에 대해 주의할점.

  • Callback Function은 일반 Function과 달리 개발자 마음대로 정의하고 호출할 수 없다. 위 예제의 Callback Function도 이미 beer라는 Array에서 정의된 대로 1) 매개변수 순서를 정의하였고, 2) 호출도 개발자가 아닌 Array 객체에서 정한 시점에 호출한다. 3) 그리고 this를 다른 포스팅에서 다루겠지만 this에 대한 바인딩도 새로 정의할 수 있다. call(), apply() 처럼.

한마디로 Callback Function은 호출한 무언가가 Callback Function에 대해 제어권 을 갖는다.
