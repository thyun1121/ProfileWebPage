베르트랑 공준

문제

https://www.acmicpc.net/problem/4948

접근법

백준 1978번 문제의 해결법처럼 (입력+1)에서 (입력*2)까지 반복문 돌리면서, 그 수들이 소수인지를 판별하기 위해 2부터 쭉 반복문을 돌리는… 이중 for문을 쓰자니 머리가 아파온다…
그래서 인터넷에서 자료를 찾아보니 에라토스테네스라는 알고리즘이 있었다.

에라토스테네스의 접근
자연수 N이 주어졌다고 가정할 때, 이 수가 소수이기 위한 필요충분조건은 N의 제곱근보다 같거나 작은 소수들로 나눠지지 않는다 이다. 그 이유는 N을 어떤 수로 나누게 되면 몫 이라는 것이 생기는데 몫과 나누는 수 둘 중 하나는 반드시 N의 제곱근 이하이기 때문이다.

즉, 위의 내용을 한마디로 말하자면,
for(int i=2; i<=sqrt(N); i++) 만큼 반복문을 실행시켜, 나눠 떨어지는 수가 있으면 소수가 아니라는 것이다.

에라토스테네스의 체
위의 에라토스테네스 접근 방식 + “체” 라는 것이 더해진 방법인데. "체"라은 단어가 붙었다시피, 무언가를 거른다는 의미이다.
자연수 N이 주어졌을 때, 2부터 N-1까지 2를 제외한 2의 배수들을 모두 체로 거른다. 그리고 남은 수 중 3을 제외한 3의 배수들을 다시 체로 거른다. 이런식으로 N의 제곱근까지 반복하여 남은 수들이 N까지의 소수가 된다.

그렇다면 에라토스테네스의 접근과 에라토스테네스의 체는 언제 구분하여 쓰면 될까?
전자는 주어진 수가 소수인지 여부를 판별할 때 사용하면 되며,
후자는 주어진 수까지의 모든 소수를 구할 때 사용하면 된다.

이제 이 문제를 풀기 위한 접근법은 다 알게되었다.
우리는 우선 에라토스테네스의 체를 이용하여 소수들을 구한 다음, 각 입력된 숫자의 범위까지 소수가 몇개 있는지 출력해주면 된다.

package baekjoon;

import java.util.Scanner;

public class BJ_4948 {

	public static void main(String[] args) {
		final int MAX = 246912;
		boolean[] notPrimeArry = new boolean[MAX+1];
		findPrimes(notPrimeArry);
		
		Scanner sc = new Scanner(System.in);
		int n;
		while(true){
			n=sc.nextInt();
			if(n==0)
				break;
			
			int cnt = 0;
			for(int i=n+1; i<=2*n; i++){
				if(!notPrimeArry[i])
					cnt++;
			}
			System.out.println(cnt);
		}
	}
	
	public static void findPrimes(boolean[] notPrimeArry){
		notPrimeArry[0]=true;
		notPrimeArry[1]=true;
		
		for(int i=2; i<=notPrimeArry.length-2; i++){
			for(int j=i*2; j<=notPrimeArry.length-1; j+=i){
				if(notPrimeArry[j]==false)
					notPrimeArry[j] = true;
			}
		}
	}

}
