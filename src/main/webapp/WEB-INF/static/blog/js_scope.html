정의

Scope라는 것은 범위를 의미하는데, 여기서 말하는 범위란,
어떤 변수들에 접근할 수 있는 범위 즉, 변수 또는 함수가 어디에서 선언되었는지 파악하여 그 대상들이 참조가 가능한 범위에 있는지를 조사하기 위해 사용되는 것이 Scope이다.

Javascript Scope에는 크게 세 가지가 있으며 Local Scope에는 또 두 종류로 세분화 할 수 있다.

  • Global Scope
  • Local Scope
    • Block Scope
    • Function Scope
  • Lexical Scope

Global Scope

변수가 함수 또는 {} 바깥에 선언되어 있는 경우, Global Scope에 정의되었다고 말할 수 있다. 이 변수는 코드 어디에서든지 참조가 가능하다. 웹 브라우저인 경우, Window객체의 프로퍼티라고도 볼 수 있다.

아래 코드 (위에서 문제로 사용된 Example2)에서 callX()함수 바깥에 선언된 var x가 Global Scope 영역에 속한다.

var x = 0;
function callX() {
  console.log(x);
  var x = 1;
  console.log(x);
}
console.log(x);
callX()
console.log(x);
  • Global Scope가 코드 어디에서든지 참조가 가능하여 편리할 수 있겠지만, 달리 생각해보면, 의도치 않은 곳에서 새로운 값으로 재할당이 될 확률이 높기 때문에 Global 전역 변수는 웬만하면 사용하지 않는 것이 좋다. (Side Effect를 줄여야 한다.)

Local Scope

함수 또는 {} 내부에 선언된 변수들의 영역을 Local Scope라고 한다.
위에서 살짝 언급했듯이 Block Scope와 Function Scope 두 가지가 존재한다.

Block Scope는 함수가 아닌 {} 내에서 유효한 Scope를 의미한다. 예를들면 if, for, while 등도 Block Scope라고 볼 수 있다.
일반 Java나 C언어 등은 Block Scope 원리를 따르지만 JavaScript는 기본적으로 Function Scope를 규칙으로 삼고 있다. (ES6부터는 let/const를 사용하면 Block Scope도 가능해짐.)
Function Scope는 함수의 {} 내에서만 유효한 Scope를 의미하며, 자세히 풀이하자면 함수 안에 선언된 변수는 해당 함수에서만 유효하고 함수 외부에서는 참조할 수 없음을 의미한다.
위에 내가 제시한 Example3와 Example4가 Function Scope와 Block Scope와 관련된 예제이다.

Function Scope Example

위 Example 3에서 살짝 변경하여 callX()내부에도 var x를 선언하였다.

var x = 0;
function callX() {
  console.log(x);
  var x = 1;
  console.log(x);
}
console.log(x);
callX()
console.log(x);

0
undefined
1
0

Function Scope는 기본적으로 함수 내부에 선언된 변수만 유효하고 외부에서는 참조할 수 없으므로, callX()에 x를 새로 선언 한 이상 함수 내부의 x는 외부와 별도로 동작하게 된다.
위의 Example3는 callX()에 x를 새로 선언하지 않았기 때문에 함수 내부의 x가 외부에 선언된 x를 참조한 것이어서 결과가 다르게 나옴을 인지하자!

Block Scope Example

위 Example4를 그대로 가져왔다.

var x = 10;
{
  console.log(x);
  var x = 20;
  console.log(x);
}
console.log(x);

let y = 10;
{
  //console.log(y);	//===> 에러. y is not defined
  let y = 20;
  console.log(y);
}
console.log(y);

위에서 문제를 풀고 답을 봐서 알겠지만 var를 사용했을 때와 let을 사용했을 때 결과가 다르다.
우선 var로 선언한 경우, Javascript는 Function Scope를 사용하므로, 변수 x가 Block 내부에서 선언되었더라도 함수가 아니기 때문에 외부에서 블록 내부를 참조할 수 있다. 하지만 ES6부터 let을 사용하면 Block Scope가 실행되어 해당 {} 영역 안에서만 변수들을 접근할 수 있다.

아래 예제도 알면 좋다. 왜 나는 지금껏 이걸 모르고 C언어처럼 써왔는지… 자괴감이 든다…

if (true) {
  var x = 100;
}
console.log(x);	//100 출력.

Lexical Scope

Lexical의 사전적 의미는 “어휘의 본질” 이라는 뜻이다. go, went, gone들의 뜻이 시점에 따라 조금씩은 다르더라도 “가다” 라는 본질을 공유하고 있듯이, Javascript에서의 Lexical Scope도 마찬가지로 "오로지 작성된 '코드’의 본질에 근거하여 참조할 수 있는 범위를 정하겠다"라는 뜻으로 파악하면 될 것 같다.
앞서 Javascript Scope의 정의를 파악할 때 코드의 실행시점에 Scope가 결정되는 것1이 아니라 작성되어 있는 코드의 문맥을 근거로 변수/함수가 선언된 영역에 따라 Scope가 결정된다고 하였다. 이것이 Lexical Scope이며 다른 말로는 Static Scope라고 한다.

두 예제를 들어볼테니 두 예제의 차이를 생각해보자.

var x = 10;

function inside() {
  console.log(x);
}

function outside() {
  x = 1000;
  inside();
}

outside();

var x = 10;

function inside() {
  console.log(x);
}

function outside() {
  var x = 1000;
  inside();
}

outside();

우선 공통적으로 Javascript는 함수 외부에서 내부로 참조하지 못한다는 것을 인지해야 한다.
두번째 코드에서 outside() 함수 내부의 x변수를 새로 선언했다. 즉, 두번째 코드는 outside() 의 x는 전역변수와 전혀 다른 새로운 변수로써, 해당 함수 안에서만 참조가 가능하다. 반면 첫번째 코드는 전역 x 변수를 모두 참조하기 때문에 출력되는 값도 outside() 함수 내부에서 재할당한 값으로 출력된다.

  1. 코드가 실행 또는 프로그램이 런타임되는 과정에서 실행 컨텍스트에 의해 정해지는 Scope를 Dynamic Scope라 한다. Perl이 이 방식을 사용하는 것 같다. ↩︎
