팩토리얼 0의 개수

원래는 ‘조합 0의 개수’ 문제를 풀고 있었는데 이 문제의 연장선상인것 같아 이 문제를 먼저 풀게 되었다.

문제

https://www.acmicpc.net/problem/1676

접근법

팩토리얼의 결과에서 뒤에서부터 0이 아닌 숫자가 나올때까지의 0의 개수를 구하는 것이 문제이다.
물론 팩토리얼을 구한 뒤 그냥 뒤에 있는 0의 개수를 구하면 간단히 끝나겠지만, 경험상 이렇게 풀게되면 timeLimit이 나올 거다. 아니면 30팩토리얼 이상을 구하게 되면 int의 범위를 넘게되어 java 같은 경우 BigInteger 또는 BigDecimal을 이용해야하는데 이게 쓰기 편치는 않으니까 PASS!!

그렇다면 수학적으로 접근해야 한다. 뒤의 0의 개수는 10이 얼마나 곱해졌는지를 의미하는 것임. 그러므로 10이 몇개인지를 정확히 알려면 10의 약수인 2와 5의 개수를 각각 구해야하며, 그 중 적은 수가 이 문제의 답이 될 것이다.

중요
5의 개수는 어떤 방법으로 구할까?
제일 쉬운 방법은 주어진 수가 1이 될때까지 for문을 돌려서 5의 배수인지 (n%5==0) 체크하며 개수를 구할 수 있다.
하지만 좀 더 효율적인 방법 을 찾아보자.
50이 주어졌을 때,
5의 배수는 50/5로 구할 수 있다.
그치만 여기서 간과한 것은 25와 그 배수일 때.
왜냐하면, 25에는 5가 더 사용되기 때문이다.
그러므로 50/5+50/25 이라는 공식을 사용하면 조금 더 효율적으로 풀이할 수 있다.
2의 배수도 마찬가지.

풀이

java


import java.util.Scanner;

public class BJ_1676 {
	public static int cntFive = 0;
	public static int cntTwo = 0;
	public static void main(String[] args) {
		Scanner sc = new Scanner(System.in);
		int n = sc.nextInt();
		System.out.println(Math.min(countFive(n), countTwo(n)));
	}
	
	public static int countFive(int n){
		for(int i=5; n/i>=1; i*=5){
			cntFive += n/i;
		}
		return cntFive;
	}
	
	public static int countTwo(int n){
		for(int i=2; n/i>=1; i*=2){
			cntTwo += n/i;
		}
		return cntTwo;
	}
}
