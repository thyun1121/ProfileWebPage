Javascript 기초부터 심화까지 공부해보자

Posted by thyun.ahn on November 13, 2018

Hoisting

console.log(timesTen(5));

function timesTen(num){
  return num*10;
}

console.log(timesHundred(5));
var timesHundred = function(num){
  return num*100;
}

자, 위의 코드는 어떤 결과가 나올까?
답은,

50
TypeError: timesHundred is not a function

읭??? 모르겠다고??
위의 예제는 hoisting과 관련있다. hoisting의 사전적 의미부터 알아보자.
우선 hoist는 “끌러올리기” 라는 뜻을 가지고 있다.
그렇다면 Javascript에서 대체 뭘 끌어올린다는 것인가?
바로 변수 선언부와 함수 선언부를 맨 위로 끌어올린다.
Javascript 엔진은 코드를 scan함과 동시에 선언부들을 제일 위로 끌러올린다.
그럼 실제로는 아래의 코드와 같이 동작하게 된다.

function timesTen(num){
  return num*10;
}
var timesHundred;

console.log(timesTen(5));
console.log(timesHundred(5));

timesHundred = function(num){
  return num*100;
}

위의 코드를 보면 timesTen은 선언부로 올라간 반면, timesHundred는 변수만 올라갔다.
hoisting은 앞서 언급했듯, 선언부만 올리기 때문에 함수 선언식인 timesTen 함수는 올라가고 timesHundred는 할당한 함수를 제외한 선언된 변수만 올라갔다.
timesHundred와 같은 함수를 함수 표현식이라고 한다.
