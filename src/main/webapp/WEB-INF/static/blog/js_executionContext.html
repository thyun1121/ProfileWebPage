Javascript 기초부터 심화까지 공부해보자

Posted by thyun.ahn on November 13, 2018

Execution Context

우리말로는 실행 컨텍스트.
실행 컨텍스트를 통해 Javascript가 어떻게 동작하고 실행되는지를 파악할 수 있기 때문에 매우 중요하다.

실행 컨텍스트를 한마디로 정의하자면, “Javascript 가 실행되는 코드 덩어리로써, 실행에 필요한 정보를 가지는 추상적인 개념”

필요한 정보는 다음과 같이 있다.

  • variable : 전역, 지역, 매개, 객체 property
  • function : 선언 정보
  • hoisting
  • scope : 변수의 유효범위
  • this : this에 binding 되는 정보
  • closure

실행 컨텍스트의 종류는 3가지가 있다.

  1. 전역 코드

    • 전역 영역에 존재하며 전역 객체의 실행 코드
    • 전역 객체는 프로그램에서 1개만 존재
      • 웹 브라우저 : window 객체
      • NodeJs : global 객체
    • 프로그램이 실행되고 전역 코드로 컨트롤이 접근하면 전역 실행 컨텍스트에 관련 실행 정보들이 객체에 담긴다.
      • 객체 : 전역객체, ex) Window
      • 실행 정보 : 변수객체, 스코프 정보 생성 및 초기화, this 바인딩
    • 프로그램 실행
    • 프로그램 종료 시 전역 실행 컨텍스트 파기

  2. 함수 코드

    • 함수의 실행코드
    • 함수 호출 시 함수의 실행 컨텍스트가 실행되면서 함수와 관련된 실행 정보들이 담긴다.
    • 함수 실행
    • 함수 실행 종료 시 함수 실행 컨텍스트가 파기

  3. eval 코드

예제코드

console.log("전역 실행 컨테스트");

func1();

function func1(){
	console.log("함수 실행 컨텍스트 1");
	func2();
}

function func2(){
	console.log("함수 실행 컨텍스트 2");
}

위의 코드의 실행 컨텍스트 순서는 다음과 같다.

  1. 전역 실행 컨텍스트 생성
  2. func1 함수 실행 컨텍스트 생성
  3. func2 함수 실행 컨텍스트 생성
  4. func2 함수 실행 컨텍스트 파기
  5. func1 함수 실행 컨텍스트 파기
  6. 전역 실행 컨텍스트 파기

즉, stack(LIFO : Last In First Out, 후입선출)으로 실행되고 파기됨을 알 수 있다.
