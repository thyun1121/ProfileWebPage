Primitive Type & Reference Type

만약 누군가 해당 주제에 대해서 물어본다면, 난 과연 제대로 답할 수 있을까? 아니! ㅋㅋㅋ
아마 “전자는 메모리에 직접 값을 할당, 후자는 메모리 주소를 할당하여 값을 참조한다.” 라며 어설픈 답을 늘어놨을 것이다.

그래서 제대로 답할 수 있도록 정리를 해보자.

Primitive Type

한글로는 ‘기본 타입’ 이라하며, 이들의 값은 ‘변경 불가능한 값’(immutable value) 이다. 그리고 메모리에 값을 그대로 할당한다.

immutable value
변경 불가능하다는 것은 해당 변수의 값을 못 바꾼다는 말이 아니라, 메모리 영역에서의 변경이 불가능하다 라는 뜻. 당연히 재할당은 가능.

아래 6가지 타입을 포함하고 있다.

  • Number
  • String
  • Boolean
  • null
  • undefined
  • Symbol (ES6 추가)

메모리 동작

var a = 100;
a = 'hundred';

  1. 데이터가 담길 공간을 확보. 즉, 0X901 메모리 확보 후 변수 a와 매칭. 현재 단계에서는 그림 상의 @0X902가 아닌 @0X901이 a와 매칭된다. 그리고 0X901 메모리에 100을 할당한다.
  2. 동일하게 'hundred’가 담길 공간 확보 (0X902) 후 값 할당한다. 그리고 변수 a를 새로운 메모리 주소 0X902와 매칭한다.
  3. 0X901 메모리의 값은 나중에 GC(Garbage Collector)에 의해서 사라지게 된다.

Reference Type

한글로는 ‘참조 타입’ 이라하며, 이들의 값은 ‘변경 가능한 값’(mutable value) 이다. Primitive Type과 다른 점은 값이 들어있는 주소값을 할당한다.

기본적으로 Object가 포함되어 있으며 Object에는 다음과 같이 존재한다.

  • Array
  • Function
  • RegExp
  • Map (ES6 추가)
  • WeakMap (ES6 추가)
  • Set (ES6 추가)
  • WeakSet (ES6 추가)

메모리 동작

var obj = {
	'name' : 'thyunAhn'
	,'career' : 4
};

var obj2 = obj;

  1. 데이터가 담길 공간을 확보. 즉, 0X904 메모리 확보. obj의 값은 {} 객체이므로 객체가 담길 또 다른 메모리 공간을 확보한다(0X903). name과 career key의 값은 primitive type이므로 각각 메모리 확보 후 값을 할당한다. (0X901 : 4, 0X902 : thyunAhn)
  2. 0X904 메모리는 0X903을 참조한다.
  3. obj 변수와 0X904 메모리 주소를 매핑한다.
  4. obj2 변수에 obj 값을 할당하므로 obj2와 매핑되는 메모리에는 그대로 obj가 바라보는 메모리 주소인 0X904가 들어간다.
  5. 이것은 결국 obj의 값이 바뀌게 되면 obj2의 값 역시 동일하게 변경됨을 의미한다.

nested 객체도 마찬가지이다.
nested 객체 인 경우 위의 reference type의 메모리 동작 과정이 한번 더 내부적으로 이루어진다.

nested 객체.
nested 객체는 아래처럼 객체 안에 객체가 있는 경우를 의미한다.

var thyun = {
 	'language' : ['javascript', 'java', 'and so on..'] 
};
