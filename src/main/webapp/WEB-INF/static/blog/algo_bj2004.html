조합 0의 개수

‘팩토리얼 0의 개수’ 백준 1676번 연장선상의 문제이다.

문제

https://www.acmicpc.net/problem/2004

접근법

조합의 공식을 생각하면 쉽게 접근할 수 있다.

nCr = n!/r!(n-r)!

이제 우리는 위의 조합 공식을 이용하여 ‘팩토리얼 0의 개수’ 백준 1676번을 상기시켜 풀면 된다.

위의 공식을 이용하여 팩토리얼의 0의 개수를 구하면 아래와 같다.

2a⋅5b(2c⋅5d)(2e⋅5f) \frac { { 2 }^{ a }\cdot { 5 }^{ b } }{ { (2 }^{ c }\cdot { 5 }^{ d })({ 2 }^{ e }\cdot { 5 }^{ f }) } (2c⋅5d)(2e⋅5f)2a⋅5b​

결국,
min(a-(c+e), b-(d+f)) 으로 이 문제를 해결 할 수 있을 것이다.

풀이

java

import java.util.Scanner;

public class BJ_2004 {
	public static int[] saveTwo = new int [3];
	public static int[] saveFive = new int [3];
	
	public static void main(String[] args) {
		Scanner sc = new Scanner(System.in);
		long n = sc.nextInt();
		long m = sc.nextInt();
		
		count(n, 0);
		count(m, 1);
		count(n-m, 2);
		
		System.out.print(Math.min(saveTwo[0]-(saveTwo[1]+saveTwo[2]), saveFive[0]-(saveFive[1]+saveFive[2])));
	}
	
	public static void count(long N, int index){
		int cntFive = 0;
		int cntTwo = 0;
		
		for(long i=5; i<=N; i*=5){
			cntFive+=N/i;
		}
		saveFive[index] = cntFive;
		
		for(long i=2; i<=N; i*=2){
			cntTwo+=N/i;
		}
		saveTwo[index] = cntTwo;
	}
}

참고
백준에 제출 시 두가지 참고해야 한다.

  • 입력 숫자의 범위
    • int 로 받게 되면 2,000,000,000이 입력되면 연산과정에서 그보다 더 큰 수가 나타날 수 있으므로, int 보단 long type으로!
  • 불필요한 연산 제거
    • 1676번 문제처럼 for문의 범위를 N/i>=1와 같이 하면 불필요한 연산으로 메모리를 많이 차지하므로, 아예 틀렸다고 채점한다. 그렇기 때문에 i<=N 와 같이 해야 정답 채점을 받을 수 있다. 휴…
