사용할수록 새로운 Java

Posted by thyun.ahn on May 22, 2018

Java

Java는 웬만하면 다 사용할 수 있다고 생각하였지만, 최근 회사 업무와 개인적으로 알고리즘 공부를 하면서 알게된 새로운 문법들이 적지않다.
역시 다시한번 틈틈이 공부해야함을 느낀다.

Reflection

reflection은 Java 기능으로써, Java program을 실행시켜 해당 프로그램을 조사하거나 스스로를 살펴볼 수 있다. 또한 프로그램의 내부 property까지 조작할 수 있다.

나같은 경우 현재 회사에서 transaction 태우기 전, reflection을 사용하여, 파라미터의 프로퍼티를 reflection을 통해서 VO객체에 제대로 들어있는 놈인지를 조사한다. 그리고나서 파라미터 값을 바인딩하여 개발자에게 콘솔로 sql 로그를 보여주고 있다.

JavaBeans에서는 빌더 툴을 사용해서 소프트웨어 컴포넌트를 시각적으로 조작할 수 있는데, 이 기능 역시 reflection을 통해서 구현한 것이라고 한다.

C,C++에서는 지원하지 않음.

예제를 통해서 어떻게 동작하는지 알아볼 필요가 있다.

Example

1) Simple Example

package reflection;

import java.lang.reflect.Method;
import java.util.Stack;
//간단예제
public class Simple {
	public static void main(String args[]){
		try{
			Class c = Class.forName("java.util.Stack");
			Method[] m = c.getDeclaredMethods();
			for(int i=0; i<m.length; i++){
				System.out.println(m[i].toString());
			}
		}catch(Throwable e){
			System.err.println(e);
		}
	}
}

위 코드의 실행결과는 다음과 같다.

public synchronized int java.util.Stack.search(java.lang.Object)

2) instanceof 연산자 흉내내기

Class.isInstance 메서드를 사용하면 instanceof 연산자를 흉내낼 수 있다.

package reflection;

class A{} 

public class InstanceSample {
	public static void main(String args[]){
		try{
			//Class cls = Class.forName("A");  //=> 에러
			Class cls = Class.forName("reflection.A");
			boolean b1 = cls.isInstance(new Integer(37));
			System.out.println(b1);
			boolean b2 = cls.isInstance(new A());
			System.out.println(b2);
		}catch(Throwable e){
			System.err.println(e);
		}
	}
}

3) Class의 method 조사

import java.lang.reflect.Method;

public class MethodSample {
	private int f1(Object p, int x) throws NullPointerException{
		if(p==null){
			throw new NullPointerException();
		}
		return x;
	}
	public static void main(String args[]){
		try{
			Class cls = Class.forName("reflection.MethodSample");
			Method[] mList = cls.getDeclaredMethods();
			for(int i=0; i<mList.length; i++){
				Method m = mList[i];
				System.out.println("name = "+ m.getName());
				System.out.println("decl class = "+m.getDeclaringClass());
				
				Class[] pList = m.getParameterTypes();
				for(int j=0; j<pList.length; j++){
					System.out.println("param #"+j+" "+pList[j]);
				}
				
				Class[] eList = m.getExceptionTypes();
				for(int k=0; k<eList.length; k++){
					System.out.println("exec #"+k+" "+eList[k]);
				}
				
				System.out.println("return type = "+m.getReturnType());
				System.out.println("===========================");
			}
		}catch(Throwable e){
			System.err.println(e);
		}
	}
}

4) 생성자 정보 얻기

//생성자 정보 파악

5) Class field 정보 얻기

package reflection;

import java.lang.reflect.Field;
import java.lang.reflect.Modifier;

public class ClassFieldSample {
	private double d;
	public static final int i=37;
	String s = "testing";
	
	public static void main(String args[]){
		try{
			Class cls = Class.forName("reflection.ClassFieldSample");
			
			Field[] fieldList = cls.getDeclaredFields();
			
			for(int i=0; i<fieldList.length; i++){
				Field fld = fieldList[i];
				System.out.println("name = "+fld.getName());
				System.out.println("decl class = "+ fld.getDeclaringClass());
				System.out.println("type = "+fld.getType());
				
				int mod = fld.getModifiers();
				System.out.println("modifiers = "+Modifier.toString(mod));
				System.out.println("==============");
			}
		}
		catch(Throwable e){
			System.err.println(e);
		}
		
	}
}

5) 이름으로 method 호출

package reflection;
import java.lang.reflect.Method;

public class InvokeMethod {

	public  int add(int  a, int  b){
		return  a+b;
	}

	public static void main(String[] args) {
		try {
			Class cls = Class.forName("reflection.InvokeMethod");
			
			Class paraTypes[] = new Class[2];
			paraTypes[0] = Integer.TYPE;
			paraTypes[1] = Integer.TYPE;
			
			Method meth = cls.getMethod("add", paraTypes);

			InvokeMethod im = new InvokeMethod();

			Object[] argList = new Object[2];
			argList[0] = new Integer(37);
			argList[1] = new Integer(43);

			Object retObj = meth.invoke(im, argList);

			Integer retVal = (Integer) retObj;
			System.out.println(retVal);
		} catch (Throwable e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
	}
}

실행결과

80

  • getMethod()를 사용하면 주어진 이름과 인자를 가진 함수를 찾을 수 있다. 이 method를 찾아서 Method객체에 저장 후, 적절한 객체 인스턴스로 해당 method를 호출한다. 또한, method를 호출하기 위해서는 파라미터 목록이 생성되어져야 한다.

6) 새로운 객체 생성하기 (= 생성자 호출하기)

package reflection;
import java.lang.reflect.Constructor;

public class CreateInstance {
	public CreateInstance(){
	}

	public CreateInstance(int a, int b){
		System.out.println(a+" : "+b);
	}

	public static void main(String[] args) {
		try {
			Class cls = Class.forName("reflection.CreateInstance");

			Class paramTypes[] = new Class[2];
			paramTypes[0] = Integer.TYPE;
			paramTypes[1] = Integer.TYPE;

			Constructor ct = cls.getConstructor(paramTypes);

			Object argList[] = new Object[2];
			argList[0] = new Integer(37);
			argList[1] = new Integer(43);

			Object retObj = ct.newInstance(argList);
		} catch (Throwable e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
	}
}

실행결과

37 : 43

  • 생성자를 호출한다는 것은 객체를 새로 생성한다는 뜻이다. (더 정확하게 말하면, 객체를 생성 시, 메모리 할당과 동시에 새 객체가 생성된다.) 생성자를 호출하는 일은 method 호출하는 것과 다르지만, 굳이 위의 method 호출과 비슷하게 예제를 만든 것이라고 한다.
  • 위의 예제는 동적으로 객체를 생성한다는 것에 의의가 있으며, 해당 생성자를 컴파일 시간이 아닌 실행시간에 호출한다.

7) 필드의 값 변경하기

package reflection;
import java.lang.reflect.Field; 

public class ChangeFieldVal {
	public  double  d;
	public static void main(String[] args) {
		Class cls;
		try {
			cls = Class.forName("reflection.ChangeFieldVal");
			Field fld = cls.getField("d");
			ChangeFieldVal cfv = new ChangeFieldVal();
			System.out.println(cfv.d);
			fld.setDouble(cfv, 12.34);
			System.out.println(cfv.d );
		} catch (Throwable e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
	}
}

실행결과

0.0
12.34

  • 리플렉션으로 객체의 데이터 필드를 변경할 수 있으며, 이 방식도 본질적으로 동적으로 동작하는 것에 가치가 있다.

8) 배열 사용하기

package reflection;
import java.lang.reflect.Array;

public class SetArray {
	public static void main(String[] args) {
		Class cls;
		try {
			cls = Class.forName("java.lang.String");
			Object arry = Array.newInstance(cls, 10);
			Array.set(arry, 5, "Testing");
			String s = (String)Array.get(arry, 5);
			System.out.println(s);
		} catch (ClassNotFoundException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
	}
}

실행결과

Testing

  • 배열이란 java에서 class의 특별한 type으로, 배열에 대한 참조는 Object 참조에 할당할 수 있다.
package reflection;
import java.lang.reflect.Array;

public class SetArray2 {  
	public static void main(String[] args) {
		int[] arrInit = new int[]{5,10,15};
		Object arry = Array.newInstance(Integer.TYPE, arrInit);
		Object arrObj = Array.get(arry, 3);
		arrObj = Array.get(arrObj, 5);
		Array.setInt(arrObj, 10, 37);
		int[][][] arrcast = (int[][][])arry;
		System.out.println(arrcast[3][5][10]);
	}
}

실행결과

37

  • 바로 위 예제는 5 X 10 X 15 차원의 배열을 만든 후, [3][5][10] index에 37이라는 값을 할당한다.
  • 한가지 유심깊게 살펴볼 것은, 컴파일 시간에 배열의 타입이 정해지지 않아도 되는 부분이다

결론

reflection을 사용하면, class의 구조를 동적으로 가져와 활용할 수 있다. 또한, java 프로그램을 실행하는 동안, class를 조작할 수 있다는 점에서 큰 의미가 있다.
